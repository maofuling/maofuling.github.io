什么是回流

回流，亦称为重排，当dom变化影响元素的几何属性（宽、高改变等等)浏览器此时需要重新计算元素几何属性并且页面中其他元素的几何属性可能会受影响，这样渲染树就发生了改变，也就是重新构造RenderTree渲染树

回流触发条件

页面布局和元素集合属性改变军会触发回流

  • 页面初始渲染
  • 添加/删除可见DOM元素
  • 改变元素位置
  • 改变元素尺寸（宽、高、内外边距、边框等）
  • 改变元素内容（文本或图片等）
  • 改变字体大小
  • 改变窗口尺寸 (resize)
  • 激活css伪类
  • 设置style属性值
  • 查询某些属性或调用某些方法

会触发回流的属性和方法

  • clienWidth,clientHeight,clientTop,clientLeft
  • offsetWidth,offsetHeight,offsetTop,offsetLeft
  • scrollWidth,scrollHeight,scrollTop,scrollLeft
  • scrollIntoView(),scrollIntoViewIfNeeded()
  • getComputedStyle() (ie使用的是currentStyle)
  • getBoundingClientRect()
  • scrollTo()
  • width/heihgt/padding/border/margin
  • font/line-heihgt/font-weight
  • position/display/float/clear

如何解决回流

css

  • 避免使用table布局
  • 尽可能在DOM树的最末端改变class
  • 避免设置多层内联样式
  • 将动画效果应用到position属性为absolute或fixed的元素上
  • 避免使用css表达式(calc())

javascript

  • 避免频繁操作样式，最好一次性重写style属性，或者将样式列表定义为class并于一次更改class属性
  • 避免频繁操作DOM，创建一个documentFragment，在它上面应用所有的DOM操作，最后再把它添加到文档中
  • 可以现为元素设置display:none,操作结束后再把他显示出来，因为在display属性为none的元素上进行的DOM操作不会引发回流和重绘
  • 对具有复杂动画的元素使用决对定位，是他脱离文档流，否则会引起父元素及后续元素频繁回流，基本原则:把动画元素用position:absolute踢出文档流，单独渲染本部分。

移动端布局小结

预热知识

什么是viewport

viewport是视窗，即显示页面的部分，移动设备上的viewport不等同于浏览器可视区宽度(移动设备的分辨率小于桌面电脑设备)。

viewport取值

  • initial-scale:初始缩放，device-width是和设备一样
  • width:宽度
  • height:高度
  • minimum-scale:允许用户的最小缩放值
  • maximum-scale:允许用户的最大缩放值
  • user-scalable:是否允许用户进行缩放，值为’yes’或’true’时表示允许，’no’或’false’表示禁止

1px问题

定义：
在不同机型上，border:1px;会出现粗细不均的情况

  1. viewport可以一套样式适配多个机型
  2. window对象中有devicePixelRatio属性，devicePixelRatio反应css像素与设备的像素比，devicePixelRatio=物理像素/独立像素

1px原因

viewport的设备和屏幕物理分瓣率是按比例而不相同(css中的1px不等于移动设备1px,由于手机像素密度不同，devicePixelRatio=设备物理像素/css像素，例如在iphone手机上，当值为2时，css像素1px即为2px，同时devicePixelRatio有兼容问题

解决方法

  • rem适配

  • flexible.js:将viewport宽度设置为实际的设备物理宽度，ios机型 scale=1/devicePixleRatio

  • 伪类+transform实现

    • 原理:把元素的before去掉，利用:before|:after重做border，并将transform的scale缩小一半，原先元素相位定位，再做border定位
      参考资料: https://www.cnblogs.com/lunarorbitx/p/5287309.html

      https://www.jianshu.com/p/7e63f5a32636

      静态布局

      传统设计，网页元素的尺寸都是使用px作为单位，设计之后，便不会因为设备变化而去适应

  1. 特点:不论浏览器尺寸具体是多少，网页均按照自己写的样式显示，可以认为此页面是固定的，再不同设备下会出现多余背景或者显示残缺
  2. 优点：无兼容性问题，对设计师友好
  3. 缺点： 有设备适配问题

    流式布局

    一套样式，但页面宽度按照屏幕分瓣率进行适配，适应不同设备
  4. 特点：使用百分比定义宽度，高度固定，屏幕分辨率变化，元素大小改变，但是布局不变
  5. 缺点:容易因为设备的宽度差异过大，造成不能按预期显示

    自适应布局

    为每个屏幕分辨率范围去设计一个静态布局，根据分瓣率切换静态布局，但页面元素不会发生变化
  6. 特点:屏幕分辨率不同时，页面位置变化但是元素大小不变化
  7. 使用@media媒体查询切换样式

    响应式布局

    网页能够自动识别屏幕宽度，并做出调整的网页设计(一套页面适应不同设备的屏幕)
  8. 特点: 当发生变化时，页面元素样式会随之变化，
  9. 优点: 适应不同设备，
  10. 缺点：媒体查询有限，目前只适应主流宽高，会有设计版本过多，造成作业量过大的困扰。
  11. 重点: 在移动端响应式布局开发中,一般不会设置最外围盒子或者容器的宽度值,都让其自己百分比自适应

    弹性布局

    使用rem或者em单位进行相对布局，支持浏览器的字体大小调整和缩放同时正常显示，em相对于父元素的font-size，rem相对于根元素html的font-size，1rem = HTML的fontsize;在IE6~8下是无法识别的

css雪碧图

css雪碧图

  1. 定义:将一个页面涉及到的所有零星图片都包含到一张大图中,使用时通过结合定位来获取准确图片位置。是网页图片应用的一种处理手段，可以优化网络
  2. 优势:
    • 减少http请求次数
    • 减少图片大小，提升网页加载速度
    • 减轻服务压力，缩短悬停加载图片所需时间延迟(多张图片加载速度小于拼合成的图片的加载速度，)敲黑板:页面加载时，不是加载单独图片，而是加载所有组合图片
    • 减少命名难的问题
  3. 劣势:
    • 背景太小，可能出现多余背景或留白；
    • 宽屏下，高分辨率的屏幕自适应页面，图片加载不够宽，容易断裂，
    • 不是直接替换照片，利用ps测量每一个的位置，过程复杂，
    • 维护不方便，如果是不经常替换的情况还可以，若是经常更换，图标预留位置不够的情况下就必须整体都替换，重写css样式，

  4. 应用场景:

    - 对于img标签设置的内容性图片，是不能合并到雪碧图的，如果合并这些图片会影响页面可读性，语义化降低且可调整的范围小
- 对于横向和纵向都平铺的图片，也不能合并到雪碧图中。如果是横向平铺，只能将所有横向平铺的图合并成一张大图，只能竖直排列，不能水平排列；如果是纵向平铺，只能将所有纵向平铺的图合并成一张大图，只能水平排列，不能竖直排列

    推荐: https://www.cnblogs.com/xiaohuochai/p/4793421.html
    https://www.jianshu.com/p/7482dde882af
    制作工具详解:http://www.cnblogs.com/wang4517/p/4476758.html

SVG

本文有些混乱，如果需要学习此知识点，可以看本文里的链接文，我的文章基本来源于此，容我整理之后，请各位使用谢谢！

知识点总结

SVG

  1. 标签

    1. defs:引用的元素容器

    2. path:定义路径

    3. svg:创建svg文档片段

      4.use:使用URI引用一个g,svg或其他具有一个唯一的ID属性和重复的图形元素。复制的是原始的元素，因此文件中的原始存在只是一个参考。原始影响到所有副本的任何改变。

      5.symbol定义一个图像模板，其本身不会输出任何图像，只有在使用实例化后才会显示，可以在svg文档里反复使用

      恩怨纠缠-defs symbol use

      1. defs
      预定义元素使其能够在svg图像里重复使用,defs元素定义的图形不能直接显示在svg上，需要借助use才可以

      什么元素可以定义为defs

      1. g   
2. symbol
3. 任何图形元素

    2. symbol

    1. 定义可以重复使用的符号，同defs，需使用use才可以显示
    2. 什么元素可以定义在里面
      1. 动画元素
      2. 描述性元素
      3. 形状元素
      4. 结构化元素
      5. 渐变元素

    3. 专有属性

      1. preserveAspectRatio
      2. viewBox

    4. use

      1. 可以在svg图像里多次重复使用一个预定义的svg图形，g元素包含于defs元素里 实例见上两例，引用g需要给g一个id值，然后再use里使用id值
      2. 浏览器在use上应用同源策略，同时在xlink:href的路径不为跨域。
      3. 跨svg调用，即可以调用其他svg文件的元素，只要是在一个文档中
      4. 什么元素可以定义在里面
        1. 任意的动画元素
        2. 任意的描述性元素

一山二虎-优点缺点

优点:

  • 支持多色图标,不再受单色限制
  • 可通过font-size,color控制样式
  • 可利用css实现动画
  • 没有失真问题
  • 减少http请求
    缺点:
  • 兼容性差,不支持ie9以下浏览器
  • 浏览器渲染svg性能一般

字体图标

[优势]

  • 矢量化：字体是矢量格式，因此能够轻松的适配不同的设备，而不必为不同分辨率的屏幕准备不同的图片资源;
  • 轻量性：一个图标字体比一系列的图像（特别是在Retina屏中使用双倍图像）要小。一旦图标字体加载了，图标就会马上渲染出来，不需要下载一个图像。可以减少HTTP请求，还可以配合HTML5离线存储做性能优化;
  • 可扩展性：图标字体可以用过font-size属性设置其任何大小。这使您能够随时输出不同大小的图标，然而，使用位图，你必须得为每个不同大小的图像输出一个不同文件。
  • 灵活性：文字效果可以很容易地应用到你的图标上，包括颜色，阴影和翻转等效果。他们还可以在任何背景下显示。
  • 兼容性：网页字体支持所有现代浏览器，包括IE低版。[ http://caniuse.com/#feat=fontface ]

[弊端]

  • 图标字体只能被渲染成单色或者CSS3的渐变色，由于此限制使得它不能广泛使用。
  • 使用版权上有限制，有好多字体是收费的。当然也有很多免费开源的精美字体图标供下载使用。
  • 创作自已的字体图标很费时间，重构人员后期维护的成本偏高。
