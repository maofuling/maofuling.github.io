知识点总结

SVG

  1. 标签

    1. defs:引用的元素容器

    2. path:定义路径

    3. svg:创建svg文档片段

      4.use:使用URI引用一个g,svg或其他具有一个唯一的ID属性和重复的图形元素。复制的是原始的元素，因此文件中的原始存在只是一个参考。原始影响到所有副本的任何改变。

      5.symbol定义一个图像模板，其本身不会输出任何图像，只有在使用实例化后才会显示，可以在svg文档里反复使用

      恩怨纠缠-defs symbol use

      1. defs

      预定义元素使其能够在svg图像里重复使用,defs元素定义的图形不能直接显示在svg上，需要借助use才可以

      什么元素可以定义为defs

      1. g
      2. symbol
      3. 任何图形元素

    2. symbol

    1. 定义可以重复使用的符号，同defs，需使用use才可以显示
    2. 什么元素可以定义在里面
      1. 动画元素
      2. 描述性元素
      3. 形状元素
      4. 结构化元素
      5. 渐变元素

    3. 专有属性

      1. preserveAspectRatio
      2. viewBox

    4. use

      1. 可以在svg图像里多次重复使用一个预定义的svg图形，g元素包含于defs元素里 实例见上两例，引用g需要给g一个id值，然后再use里使用id值
      2. 浏览器在use上应用同源策略，同时在xlink:href的路径不为跨域。
      3. 跨svg调用，即可以调用其他svg文件的元素，只要是在一个文档中

    5. 什么元素可以定义在里面

      1. 任意的动画元素
      2. 任意的描述性元素

        一山二虎-优点缺点

        优点:
  • 支持多色图标,不再受单色限制
  • 可通过font-size,color控制样式
  • 可利用css实现动画
  • 没有失真问题
  • 减少http请求
    缺点:
  • 兼容性差,不支持ie9以下浏览器
  • 浏览器渲染svg性能一般

  1. 制作svg Sprite

    1. icon获取(icomoon.io,阿里的iconfont.cn,bootsrap的font-awesome):从以上官网里直接下载，找到你想要的图标点击然后出现下载图标，然后出现SVG,你接着点击SVG，就可以下载了
    2. 合并SVG，使用gulp
      1. npm install grunt-svgstore –save-dev
      2. 按着这个文件夹顺序（node_modules->grunt-svgstore)找到Gruntfile.js,打开它在里面加上”grunt.loadNpmTasks(‘grunt-svgstore’);”.svg图标颜色使用fill填充,cleanup为true
      3. 在svgstore:{}里配置好,并切换到grunt-svgstore目录,在命令行里输入grunt,到此基本结束
